웹서버? 클라이언트?


저희가 이때까지 만들어온 것은 웹페이지였습니다, 그리고 그것은 파일이죠. 이제 다른 사람들한테는 어떻게 보여줄까요?

그러려면 먼저 인터넷이 어떻게 동작하는지 알아야합니다.



인터넷은 컴퓨터가 최소 2대 이상 있을 때부터 구성됩니다. 그리고 컴퓨터들끼리 상호작용을 합니다.

위 사진처럼 정보를 주는 컴퓨터를 서버 컴퓨터라고 하며, 정보를 요청하고 받는 컴퓨터를 클라이언트라고 합니다.


이제 저희는 클라이언트였던 입장에서 벗어나 서버 컴퓨터를 구성하는 방법에 대해 알아보겠습니다.

웹 서버


서버를 어렵다고 생각할 수 있지만, 클라이언트 컴퓨터들에겐 웹을 열수 있는 브라우저 프로그램이 있듯이, 서버도 단순히 서버를 여는 프로그램을 설치하면 됩니다.



저희는 그 중 아파치를 사용해볼 것입니다.

아파치는 웹 서버 시장에서 가장 많이 사용되는 부동의 제왕입니다.

Bitnami


사용하는 운영체제가 윈도우이므로, 윈도우 기준으로는 Bitnami WAMP Stack이라는 프로그램을 이용하면 손쉽게 아파치 서버를 설치할 수 있습니다.


W는 Windows, A는 Apache, M은 MySQL, P는 PHP의 약자인데 이 프로그램을 이용하면 이후 백엔드의 구축을 보다 쉽게 할 수 있습니다.



설치 이후에는 이렇게 간단하게 웹서버를 온/오프 할 수 있습니다.

HTTP


웹 서버를 구성했다면, 우린 이제 단순히 컴퓨터에서 html 파일을 열어 우리한테만 보이던 웹을 다른 사람에게 보여줄 수 있게 되었습니다.


비트나미가 설치된 경로에 가서 htdocs라는 파일 내에 만들었던 html 파일을 넣어주면 됩니다.

그리고 서버를 실행하면, 웹 브라우저의 주소가 바뀝니다. 우리 컴퓨터 내의 경로에서 아이피 주소로요.

사용자의 입장에서는 아무런 변화가 없지만, 내부적인 프로세스는 완전히 달라졌습니다.

http://는 HyperText Transfer Protocol의 약자입니다. Transfer는 전송이라는 뜻입니다. 즉 웹페이지를 전송하기 위해서 만든 통신 규약이라는 뜻입니다. 주소창에 입력된 주소가 http://로 시작하면 웹브라우저는 http 통신규약에 따라서 웹서버에 접속하게 됩니다. file://로 시작하면 웹브라우저는 파일을 직접 열어서 보여줍니다.

웹 호스팅

이렇게 웹 서버를 구성해보았는데, 아쉽게도 이런 기본 설정을 통해서는 서버와 클라이언트의 네트워크가 같아야합니다. 매우 기초적인 네트워크망이라는 것이죠.

자신만의 도메인을 이용해 주소를 만들어 공개적인 웹을 만들려면, 서버 컴퓨터는 항상 켜져있고, 네트워크에 좀 더 복잡한 설정을 해야 합니다.

하지만 이런 서버 시스템을 쉽게 구성하도록 도와주는 것이 있습니다. 바로 웹 호스팅입니다.

저희는 수많은 웹 호스팅 업체중, 무료이고 유명하며 신뢰성있는 Github(https://github.com/)의 gitpage기능을 이용해보겠습니다.

github에 프로젝트(repository)를 생성하고 거기에 제작한 html 파일을 업로드 한 후, setting에서 약간의 설정을 해주면 웹사이트가 생깁니다.

Github Pages 항목에서 master branch를 선택하고 Select branch 중 master branch를 선택합니다. 그리고 Save 버튼을 누릅니다.

이렇게, 웹사이트 주소가 생성됩니다.

보기에는 변함이 없지만 내부적인 구조는 이렇게 바뀝니다.

페이지의 주소를 보면 지금 현재 이 웹사이트도 이 깃허브 페이지를 이용하고 있다는 걸 알 수 있습니다.

여기까지 할 수 있게 되었다면, 축하합니다. 이제 우리가 만든 웹을 전세계 사람들에게 공유할 수 있게 되었습니다.
